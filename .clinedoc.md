# 🍀 El Palomo Negro — Plataforma de Rifas y Boletas

**El Palomo Negro** es una plataforma web integral diseñada para la gestión, reserva, venta y procesamiento de sorteos y boletas de la suerte en Colombia. El sistema está optimizado para centralizar la operación de los asesores de venta, garantizando transacciones seguras y una comunicación inmediata con el cliente final.

---

## 🛠️ Stack Tecnológico

* **Backend:** Laravel 11+ (PHP 8.2+)
* **Panel de Administración:** Filament v3 (Livewire + Tailwind CSS)
* **Interactividad en Cliente:** Alpine.js (nativo en Filament)
* **Base de Datos:** MySQL / MariaDB (Tipos de datos `DECIMAL(15,2)` para control estricto de dinero)
* **Localización:** Colombia (Formato de moneda en pesos `$`, números de celular con indicativo internacional `+57`)

---

## 🏗️ Arquitectura y Componentes Clave

### 1. Gestión de Sorteos (`RaffleResource`)
* Permite la creación, parametrización y control de las rifas activas.
* **Control Financiero Blindado:** Incluye campos de dinero para el premio mayor (`jackpot_prize`) protegidos con máscaras de miles mediante puntos, formateados nativamente en PHP para evitar la corrupción de ceros decimales en la base de datos al guardar o editar registros.

### 2. Pedidos Temporales y Reservas (`TicketOrderResource`)
* **Validación Asíncrona en Caliente:** Los asesores disponen de un botón de verificación (`suffixAction`) en el campo de números reservados que consulta en tiempo real si las boletas deseadas ya están ocupadas en la tabla definitiva de tickets o en otras órdenes pendientes, previniendo duplicados antes de guardar.
* **Timer Local de Retención:** Un badge circular grande y animado con el color corporativo que pulsa suavemente (`animate-pulse`) mediante Alpine.js, controlando localmente el tiempo restante de la reserva sin generar peticiones en bucle ni sobrecargar el servidor.

### 3. Emisión de Tickets Definitivos (`Ticket`)
* **Transaccionalidad Atómica:** Al cambiar el estado de un pedido de "Pendiente" a "Procesado" dentro del formulario, el sistema ejecuta de forma interna una transacción SQL segura (`DB::transaction`) que migra los números apartados hacia la tabla de boletas definitivas con estado de pago verificado (`PAID`).

### 4. Automatización de Notificaciones (WhatsApp Web)
* Integración fluida con la API de WhatsApp mediante los ganchos del ciclo de vida de Filament (`afterSave` / `afterCreate`).
* **Sanitización Telefónica:** Limpieza automatizada de caracteres no numéricos en el teléfono del cliente, inyectando de forma inteligente el indicativo de país `57` si el número ingresado tiene 10 dígitos.
* **Mensajería Pulida:** Uso de plantillas en formato *Heredoc* formateadas al margen izquierdo para garantizar mensajes limpios, legibles y con emojis profesionales en el chat del usuario.

### 5. Panel de Administración y Configuración (`AdminPanelProvider`)
* **Identidad Corporativa Centralizada:** Gestión de colores, logotipos, alturas visuales y nombres de marca desde un único punto, utilizando la paleta nativa de Tailwind CSS para mantener consistencia visual en toda la plataforma.

### 6. Módulo de Configuraciones en la tabla Settings (`SettingsResource`)
* Usando los parámetros key y value se creará un seeder con las key definitivas para la plataforma completa con los campos de parametrización por defectos, una única ventana para configurar tanto la landingpage o la página web y el panel de administración, incluyendo la configuración de WhatsApp, correos electrónicos y otros parámetros críticos del sistema.

### 7. Módulo de Reportes y Exportación de Datos (`ReportResource`)
* Permite la exportación de datos en formatos CSV y Excel, con filtros avanzados para generar reportes de ventas, reservas y transacciones financieras.

## 📈 Características de Interfaz de Usuario (UI/UX)

* **Ancho de Pantalla Completo:** El panel está configurado con un ancho extendido (`maxContentWidth`) para facilitar la lectura de grillas complejas y listas masivas de números.
* **Menú Lateral Colapsable:** Optimización del espacio de trabajo en computadores de escritorio (`sidebarCollapsibleOnDesktop`).
* **Sidebar con Contadores Inteligentes:** Uso de globos dinámicos (`NavigationBadge`) en color de advertencia para alertar a los asesores sobre el número exacto de pedidos pendientes por procesar. El globo se oculta automáticamente cuando la lista queda en cero.
* **Identidad Corporativa Centralizada:** Colores, logotipos, alturas visuales y nombres de marca gestionados desde el `AdminPanelProvider` utilizando la paleta nativa de Tailwind CSS.

## 🚀 Prácticas de Código Aplicadas
1.  **Cero JS invasivo:** Preferencia absoluta por características nativas de Filament v3 y Alpine.js.
2.  **Aislamiento de Clases:** Evita colisiones en PHP mediante el uso de nombres de clase absolutos en componentes interactivos flotantes.
3.  **Heredoc Limpio:** Cero sangrías internas en cadenas multilínea para asegurar la correcta interpretación de saltos de línea y espaciados en la aplicación móvil de WhatsApp.
